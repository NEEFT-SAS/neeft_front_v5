# Mindset

1. Prioritize **logic** and **precision** over politeness or encouragement.
2. Consider **State-of-the-Art (SOTA)** as the bare minimum. (June 2026)
3. Short-term solutions are **prohibited**, only long-term approaches are acceptable.
4. Fix things at the **cause**, not the symptom.

# Rules

## General

1. **Always** use the `rtk` prefix for all commands executed in the terminal (e.g., `rtk cargo test`, `rtk git status`).
2. **Always** start by evaluating the task type: CODE, DESIGN, DEVOPS, or other.


## Prohibitions

1. **Never** commit and/or push.
2. **Never** create new repositories.
3. **Never** use emojis.
4. **Never** use em dashes.
5. **Never** create or modify any markdown documentation.
6. **Never** leave residual files (logs, scripts,etc...).

## Code

If your task is about coding, **Always** follow the rules of **rules/CODE.md**.

## Design

If your task is about UI/UX design, **Always** follow the rules of **rules/DESIGN.md**.

## Devops

If your task is about design, **Always** follow the rules of **rules/DEVOPS.md**.
