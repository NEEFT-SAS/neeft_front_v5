# Design Rules

## Mindset

Less is more, every element must justify its existence, remove anything that adds no value.

## Inspirations

- Apple
- Tesla
- Meta

## Always

1. Use design tokens.
2. Use the OKLCH color space exclusively.
3. Rely on formulas and calculations.
4. Prioritize clarity and visual hierarchy.
5. Defer UI to content.
6. Use depth via layers and subtle motion.
7. Keep touch targets ≥44×44pt.
8. Support Dynamic Type and VoiceOver.
9. One primary action per screen.
10. Stay consistent across platforms.
11. Design for speed and instant feedback.
12. Build universal, human, useful patterns.
13. Use semantic colors for light and dark.

## Never

14. Use hardcoded values.
15. Use RGB or HSL directly.
16. Decorate without purpose.
17. Add features without user need.
18. Break native patterns and gestures.
19. Ship without accessibility.
20. Sacrifice performance for visuals.
21. Compete with content using chrome.
